# Daniel Rosehill

Documented AI practitioner with established online presence (`danielrosehill.com`). Maintains [[Claude Code Repos Index (Daniel Rosehill)|Claude Code Repos Index]] — **75+ Claude Code repositories** across systems administration, productivity, legal/research, health, communications, multi-agent systems, media, data work.

The index explicitly clarifies he's human, not AI-generated, addressing skepticism about prolific output.

For our wiki, Daniel matters as **proof of concept that Claude Code is a general-purpose agentic operating system, not a coding assistant**.

## Distinctive contributions

### Domain breadth
Most ecosystem authors focus on coding-adjacent domains. Daniel applies Claude Code patterns to:
- Bash alias management, KDE Plasma integration, Linux debugging
- Career tracking, budgeting, personal planning
- Evidence logging, document analysis, deep research
- Medical visit management, therapy tracking
- Audio/video editing, podcast assembly

The breadth itself is the contribution — it normalizes treating Claude Code as substrate for *any* knowledge work, not just code.

### Agent Workspace Model v1.1
Daniel's `Agent-Workspace-Generator` provides standardized scaffolding conforming to a documented "Agent Workspace Model." Validation + publishing slash commands. Worth treating as candidate convention for personal Claude Code projects.

### 28 cluster plugins
Consolidate domain primitives (commands, skills, agents) globally; provision per-project scaffolds on demand. **Scalable alternative to forking template repos** — applies one plugin's worth of conventions across many projects without per-project copy-paste.

### `Claude-Code-Linux-Desktop-Slash-Commands`
Specific to Linux desktop environments: hardware benchmarking, filesystem organization, security posture validation. Niche but useful for Linux desktop users.

### `CONTEXT.md` and `Claude-Code-Context-Toolkit`
Patterns for separating human-authored context from AI-optimized briefings. Conceptually relevant to [[Memory]] design and to [[Iterative Retrieval]].

### Claude Development Agents
74+ pre-configured agent setups for development workflows.

## What this represents

Daniel's prolific output (75+ repos, sustained over months) suggests:
1. The marginal cost of building a Claude Code project for a new domain has dropped low enough for prolific application
2. Claude Code's primitives (folder structure, CLAUDE.md, slash commands, MCP) are domain-agnostic enough to support arbitrary use cases
3. There's a mature mental model for human-AI collaboration workflows that one person can repeatedly apply

This is a bigger insight than any single repo. **Claude Code mastery is partly a mindset shift**: see new domains, ask "could I build a Claude Code workspace for this?"
